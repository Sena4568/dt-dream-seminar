# 특별 발표 데모 자료 — 안내

> "클로드 활용 예시 30~40분 발표"의 데모 영상 5개를 녹화할 때 쓰는 입력·산출물 자료입니다.
> 예시 강좌: **신입사원 보고서 작성 기초 (3차시)**

## 자료 목록

| 자료 | 파일 | 쓰이는 데모 | 역할 |
|---|---|---|---|
| 1 | `데모A_수업메모_정리전.md` | 데모 A | 일부러 지저분하게 쓴 1차시 수업 준비 메모 (입력) |
| 2 | `데모A_교안_정리후.md` | 데모 A | 클로드가 정리한 표준 교안 (결과 / 백업) |
| 3 | `데모B_프로젝트지식/` (교안 3개) | 데모 B | 1·2·3차시 교안 — 프로젝트 지식으로 업로드 |
| 4 | `데모C_복습퀴즈.html` | 데모 C | 1차시 개념 인터랙티브 복습 퀴즈 |
| 5 | `교안만들기스킬/SKILL.md` | 데모 D | '교안 만들기' 커스텀 스킬 본체 |
| 6 | `데모D_새수업메모.md` | 데모 D | 스킬 테스트용 새 수업 메모 (검토·수정 차시) |
| 7 | `데모E_수업사이트/` | 데모 E | 교안+퀴즈를 묶은 수업 사이트 (바이브 코딩 산출물) |

## 자료 안에 심어 둔 설계

발표 데모가 매끄럽게 흐르도록 자료에 두 가지를 의도적으로 넣었습니다.

- **데모 B "안 다룬 개념" 질문용**: 1~3차시 교안 어디에도 '검토·수정'을 다루는 차시가 없습니다. 1차시 교안의 '보고서 쓰기 5단계 흐름'에 단어만 등장합니다. → 클로드가 "검토·수정이 빠졌다"고 답하게 됩니다.
- **데모 B "겹치는 내용" 질문용**: '두괄식 / 핵심 먼저'가 1차시(보고서 전체 구조)와 2차시(문단 단위)에 모두 나옵니다. → 클로드가 "두괄식이 겹친다"고 답하게 됩니다.
- 데모 D의 새 메모(자료 6)가 바로 그 '검토·수정' 차시라, 데모 B에서 발견한 빈 곳을 데모 D가 채우는 흐름이 됩니다.

## 데모별 연결

데모 A의 교안 한 건이 전체를 관통합니다.

```
자료 1 (수업 메모)
  └→ 데모 A → 자료 2 (교안)
        └→ 데모 B 입력: 자료 3 (차시 교안 3개)
        └→ 데모 C 입력 → 자료 4 (복습 퀴즈)
        └→ 데모 D: 자료 5(스킬) + 자료 6(새 메모) → 교안 자동 생성
        └→ 데모 E 입력: 자료 2 + 자료 4 → 자료 7 (수업 사이트)
```

## 참고

- HTML 파일(`데모C_복습퀴즈.html`, `데모E_수업사이트/index.html`)은 더블클릭하면 브라우저에서 바로 열립니다.
- 수업 사이트는 `index.html` · `style.css` · `script.js` 세 파일로 된 작은 프로젝트입니다. 세 파일을 같은 폴더에 함께 두어야 정상 동작합니다.
- 강좌 주제는 청중 직군에 맞춰 교체 가능합니다. 교체 시 자료 1~7과 퀴즈 문항을 같은 양식으로 다시 채우면 됩니다.
